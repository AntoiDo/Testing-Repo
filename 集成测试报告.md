## 六、测试基础设施

### 6.1 隔离策略

| 文件 | 数据库 | 清理方式 |
|------|--------|----------|
| `test_integration_pipeline.py` | 临时 SQLite（`tempfile.mkdtemp`） | `tearDownModule` 删除整个临时目录 |
| `test_integration_api.py` | 临时 SQLite（`tempfile.mkdtemp`） | `tearDownModule` 删除整个临时目录 |
| `test_integration_database.py` | 临时 SQLite（`tempfile.mkdtemp`） | `tearDownModule` 删除整个临时目录 |

### 6.2 依赖处理

- **重型依赖 stub**：`ffmpeg`、`fastapi`、`blinker` 使用运行时模块替换（`sys.modules`），无需安装完整环境
- **轻量依赖直接导入**：`app.db.*`、`app.validators.*`、`app.models.*` 等直接导入使用

### 6.3 运行方式

```bash
# 运行全部集成测试
cd backend
python -m pytest tests/test_integration_*.py -v

# 运行单个文件
python -m pytest tests/test_integration_pipeline.py -v

# 运行单个类
python -m pytest tests/test_integration_pipeline.py::TestPipelineStage1URLParsing -v

# 运行单个方法
python -m pytest tests/test_integration_api.py::TestHealthCheck::test_sys_health_returns_200 -v
```

## 八、已知局限性

1. **无性能测试**：未覆盖并发请求、大文件下载、长视频转写等场景
2. **无真实外部依赖**：LLM API、youtube-dl、Whisper 均为 stub，无法验证真实网络交互
3. **仅后端覆盖**：前端 `BillNote_frontend/` 无集成测试，Extension 仅有基础 E2E
4. **无报告输出**：当前仅依赖 unittest 标准输出，未生成 JUnit XML / HTML 等可持久化报告
5. **transcriber 依赖**：当 `faster-whisper` 不可用时，部分 transcriber 测试会被 `skipTest`
